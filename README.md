## midterm problem
Your group of four was just given the go-signal to start the final project for COM 103. The topic: build a simple Python program for a real-world scenario. The group chat is already blowing up with task assignments and “sino ang gagawa ng ano.”

# Python
the python file show this action and steps
### 1
Ask for the project title and the group name.

### 2.
Display the task types using a loop.
   
### 3
Accept 4 task assignments. For each assignment, ask the task number (1–5). If the user types 0, that assignment slot is skipped.

### 4. 
If the task number is valid (1–5), ask for the member’s name and the task status: done or pending.
 
### 5.
Points per task: done = 2 points; pending = 1 point.

### 6.
After all assignments: compute total points earned, maximum possible points (2 × number of assigned tasks), and progress as a whole-number percentage (earned / max_possible × 100).
 
### 7.
Assign a project status label: progress ≥ 75% → 'PROJECT READY!'; progress ≥ 50% → 'ON TRACK.'; progress < 50% → 'NEEDS MORE WORK!'.
  
### 8.
Print a formatted project board showing: project title, group name, each assigned task (sequential number, task type, estimated hours, member name, status, points), total points, progress %, and project status.
